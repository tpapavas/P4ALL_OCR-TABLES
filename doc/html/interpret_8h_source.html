256/*
257 Create a filter processor. This filters the PDF operators
258 it is fed, and passes them down (with some changes) to the
259 child filter.
260
261 The changes made by the filter are:
262
263 * No operations are allowed to change the top level gstate.
264 Additional q/Q operators are inserted to prevent this.
265
266 * Repeated/unnecessary colour operators are removed (so,
267 for example, "0 0 0 rg 0 1 rg 0.5 g" would be sanitised to
268 "0.5 g")
269
270 The intention of these changes is to provide a simpler,
271 but equivalent stream, repairing problems with mismatched
272 operators, maintaining structure (such as BMC, EMC calls)
273 and leaving the graphics state in an known (default) state
274 so that subsequent operations (such as synthesising new
275 operators to be appended to the stream) are easier.
276
277 The net graphical effect of the filtered operator stream
278 should be identical to the incoming operator stream.
279
280 chain: The child processor to which the filtered operators
281 will be fed.
282
283 old_res: The incoming resource dictionary.
284
285 new_res: An (initially empty) resource dictionary that will
286 be populated by copying entries from the old dictionary to
287 the new one as they are used. At the end therefore, this
288 contains exactly those resource objects actually required.
289
290 The filter options struct allows you to filter objects using callbacks.
291*/
292pdf_processor *pdf_new_filter_processor(fz_context *ctx, pdf_document *doc, pdf_processor *chain, pdf_obj *old_res, pdf_obj *new_res, int struct_parents, fz_matrix transform, pdf_filter_options *filter);
